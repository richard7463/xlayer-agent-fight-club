## Mandatory Requirements

Human-track requirements called out on the official page:

1. At least one part of the project is built on X Layer for `X Layer Arena`.
2. Create an `Agentic Wallet` as the project's onchain identity.
3. Use at least one core module from `Onchain OS skills` or `Uniswap skills`.
4. Push code to a public GitHub repo with a README covering:
   - project intro
   - architecture overview
   - deployment address
   - Onchain OS / Uniswap skill usage
   - working mechanics
   - team members
   - project positioning in the X Layer ecosystem
5. Submit through the official `Google Form` before `April 15, 2026 23:59 UTC`.

## Bonus Signals

The human page explicitly says these can help:

- a `1-3 minute` demo video
- a project intro post on `X`
- effective use of `more Onchain OS skills` or `Uniswap skills`

## Scoring

The human page states four weighted dimensions:

- `Onchain OS / Uniswap integration and innovation` — `25%`
- `X Layer ecosystem integration` — `25%`
- `AI interactive experience` — `25%`
- `Product completeness` — `25%`

It also states that:

- AI agent judges scan `GitHub repos` and `onchain transaction history`
- human judges review creativity and practicality on top of the quantitative layer

## What This Means For Agent Fight Club

If this project is ever adapted for the human track, the strongest alignment is still `X Layer Arena`, not `Skills Arena`.

Why:

- it is a complete product, not a reusable module
- it has a clear AI interaction surface
- it can benefit from multiple Onchain OS modules
- it can produce onchain evidence through the fighters

## Current Gap Against The Human Track

Today the repo is stronger on:

- Moltbook-native product story
- multi-agent league structure
- runtime evidence and social loop

It is still weaker on the human page's hard quantitative expectations because it does not yet ship:

- direct Agentic Wallet routing for the season fighters
- final OnchainOS DEX transaction proofs in the app
- final public deployment URL

## Practical Takeaway

For now:

- use this repo for the `Moltbook agent track`
- treat the human track as a second-layer expansion target, not the current submission target

If later upgraded for the human page, the priority order should be:

1. direct `OnchainOS Wallet + DEX` execution
2. final public deployment
3. richer multi-skill footprint
4. stronger X / demo distribution
